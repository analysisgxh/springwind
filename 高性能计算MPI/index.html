    MPI并行计算-Parallel Programming using MPI

    安装配置MPI库

    基本概念 MPI ：

    • MPI概念: 消息传递接口（Message Passing Interface,MPI）
    • CC (Coherence and Cohesion): 连贯与衔接，段落之间的逻辑过渡。
    • MPI的目标: 实现具有高性能和可移植性的大规模并行程序。
    • MPI的介绍: MPI通过在每个节点(机器)启用一个或者多个进程来实现并行计算，每个进程还可以含有多个线程，故有时也将MPI和OpenMP结合使用。MPI

    • 参考文档:

      • OpenMP基础
      • MPI函数详情
      • 相关pdf
      • 《并行计算导论》张林波等，《并行算法实践》陈国良等。
      • 常用MPI库：intel MPI,MS MPI,OpenMPI,MPICH等。
      • 安装intel oneAPI全家桶，简单。
      • 单独安装intel MPI,运行可能遇到通信问题。
      • 使用MS MPI:
      • 安装两个文件:mpi库和执行程序mpiexec

      VS Code 配置 MS-MPI (C/C++) 环境笔记

    在 VS Code 中配置 MPI 环境，本质上是将 Visual Studio 图形界面里的核心操作（包含目录、库目录、依赖项）通过 JSON 配置文件 或 编译参数 来实现。

    1. 配置代码提示（解决红波浪线）

    对应 VS 操作： 设置 .h 文件目录 (Additional Include Directories)

    • 文件： .vscode/c_cpp_properties.json (或通过 UI 界面设置)
    • 操作：
    • 按 Ctrl + Shift + P，输入并选择 C/C++: Edit Configurations (UI)。
    • 找到 包含路径 (Include path)，添加你的 MPI include 路径。
    • 路径示例： H:/Include（如果是 Intel MPI 则类似 C:/.../mpi/latest/include）。
    • 效果： 告诉 IntelliSense 头文件在哪里，#include <mpi.h> 不再报错。

    2. 配置编译任务（核心步骤）

    对应 VS 操作： 设置 .lib 目录 (Additional Library Directories) * 填入所需的 lib 文件名 (Additional Dependencies)

    • 文件： .vscode/tasks.json
    • 操作： 在 args 数组中添加对应的编译和链接参数。

    方案 A：使用 GCC 编译器 (g++/MinGW)

    "args": [
    "-g",
    "${file}",
    "-o", "${fileDirname}\\${fileBasenameNoExtension}.exe",
    "-I", "H:/Include",                     // 1. 指定头文件目录
    "-L", "H:/Lib/x64",                     // 2. 指定库文件目录
    "-lmsmpi"                               // 3. 链接具体的库 (注意去掉 .lib 后缀)
]

    配置文件说明实例

    • c_cpp_properties.json 
      {
    "configurations": [
        {
            "name": "Win32-MPI",
            "includePath": [
                "${workspaceFolder}/**",
                "H:/MPI/Include"
            ],
            "defines": [
                "_DEBUG",
                "UNICODE",
                "_UNICODE"
            ],
            "compilerPath": "H:/MPI/mingw/winlibs-x86_64-posix-seh-gcc-15.2.0-mingw-w64ucrt-13.0.0-r6/mingw64/bin/g++.exe",
            "cStandard": "c17",
            "cppStandard": "gnu++14",
            "intelliSenseMode": "windows-gcc-x64"
        }
    ],
    "version": 4
}
    • tasks.json 
                 {
         "tasks": [
             {
                 "type": "cppbuild",
                 "label": "C/C++: g++.exe 编译 MPI 程序",
                 "command": "H:/MPI/mingw/winlibs-x86_64-posix-seh-gcc-15.2.0-mingw-w64ucrt-13.0.0-r6/mingw64/bin/g++.exe",
                 "args": [
                     "-fdiagnostics-color=always",
                     "-g",
                     "${file}",
                     "-o",
                     "${fileDirname}\\${fileBasenameNoExtension}.exe",
                     "-I",
                     "H:/MPI/Include",
                     "-L",
                     "H:/MPI/Lib/x64",
                     "-lmsmpi"
                 ],
                 "options": {
                     "cwd": "${fileDirname}"
                 },
                 "problemMatcher": [
                     "$gcc"
                 ],
                 "group": {
                     "kind": "build",
                     "isDefault": true
                 },
                 "detail": "使用 MinGW g++ 并链接 MS-MPI 库"
             }
         ],
         "version": "2.0.0"
     }

    Note

    # 注意替换为你的实际编译器路径，这里以 g++ 为例
g++ hello_mpi.cpp -o hello_mpi.exe -I"H:\Include" -L"H:\Lib\x64" -lmsmpi
mpiexec -n 4 .\hello_mpi.exe
